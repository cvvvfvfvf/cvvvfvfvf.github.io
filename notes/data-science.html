Distance

  • Properties: Non-negativity, identity (reflexivity), symmetry, triangle inequality

  • Numerical distances:

    • Minkowski distance: For two $d$-dimensional vectors $x = (x_1, x_2, \dots, x_d)$ and $y = (y_1, y_2, \dots, y_d)$: \[ D_p(x, y) = \left( \sum_{i=1}^d |x_i - y_i|^p \right)^{\frac{1}{p}}, \quad p \ge 1 \]
    • Manhattan distance: $p = 1$, Minkowski distance reduces to Manhattan distance: \[ D_1(x, y) = \sum_{i=1}^d |x_i - y_i| \]
    • Euclidean distance: $p = 2$, Minkowski distance reduces to Euclidean distance: \[ D_2(x, y) = \sqrt{\sum_{i=1}^d (x_i - y_i)^2} \]
    • Maximum distance: As $p \to \infty$, Minkowski distance reduces to Chebyshev distance: \[ D_\infty(x, y) = \max_{i=1, \dots, d} |x_i - y_i| \]
    • Cosine similarity Range is $[-1, 1]$, values closer to 1 indicate more similar direction.
      $$ \cos\theta = \frac{\sum_{i=1}^{n} A_i B_i}{\sqrt{\sum_{i=1}^{n} A_i^2} \times \sqrt{\sum_{i=1}^{n} B_i^2}}$$

  • Non-numerical distances:

    • Hamming distance: The number of positions at which the corresponding symbols are different between two equal-length strings

    • Edit distance: The minimum number of single-character insertions/deletions/substitutions required to transform one string into another

    • Jaccard distance = 1 - Jaccard similarity (the ratio of the intersection of two sets to their union)

  • One-Hot Encoding: Converts a discrete feature with a single value into a binary feature (value 0 or 1)

    Advantages: Solves the linear relationship problem among features in One-Hot encoding, avoiding the impact of feature encoding on models like linear regression

Data Standardization

Why perform data standardization?

  • Many machine learning algorithms require input features to be in standardized form

  • If the magnitude differences between features are too large, similarity assessment between samples will be biased

Methods:

  • Z-Score Standardization (applicable when the maximum or minimum of the feature is unknown or the sample distribution is very discrete) \[ f_i' = \frac{f_i - \mu}{\sigma} \]

    where $\mu$ is the mean of the feature, $\sigma$ is the standard deviation of the feature.

  • Min-Max Standardization (0-1 standardization applicable when simply transforming data to a specific interval) \[ f_i' = \frac{f_i - f_{\min}}{f_{\max} - f_{\min}}, f_i' = \frac{b-a}{f_{\max} - f_{\min}} \left( f_i - f_{\min} \right) + a \]

    Maps the feature to the interval $[a,b]$: Disadvantages: When new data is added, the maximum or minimum of the feature may change, requiring redefinition; performs poorly if the data contains outliers.

  • Sigmoid Transformation (Logistic Transformation) --- Non-linear transformation: \[ f_i' = \frac{1}{1 + e^{-f_i}} \]

Feature Discretization

Why?

  1. Specific algorithm requirements: e.g., association rule mining algoriComplexity ReductionComplexity Reductionthms, ID3 decision tree algorithm

  2. Enhance robustness: Discretization essentially segments continuous data, thus outliers are directly placed into corresponding intervals, enhancing robustness to data outliers;

  3. Improve interpretability: The values of discretized features are transformed into interval numbers with clear meaning

  4. Improve performance: Significantly reduces the number of feature values, reducing storage requirements and improving model computational efficiency

Process: Divide the value range of a continuous feature into several bins, then use the bin to replace the feature values falling within that bin.

The dividing points between intervals are called cutpoints. To create k intervals (cardinality), (k-1) cutpoints are needed.

Goal: Minimize the cardinality while minimizing information loss.

Divided into (based on whether target information of the dataset is referenced): Unsupervised discretization (Equal-width discretization, Equal-frequency discretization, Cluster-based discretization), Supervised discretization (Information gain discretization, ChiMerge discretization)

Equal-width discretization:

  • Evenly divide the feature values into k intervals, each with equal width, then assign feature values to corresponding intervals.

  • Disadvantage: Equal-width discretization has high requirements for input data quality and cannot handle the problem of outliers.

Equal-frequency discretization:

  • Does not require interval widths to be consistent, ensuring that the sample size within each discretized interval is balanced.

  • Based on the total number of continuous features, evenly divides them into k intervals, each interval containing the same number of samples, then the value range of each portion is the corresponding discretization interval.

  • Disadvantage: Sometimes same or similar samples may be divided into different intervals, potentially causing adjacent intervals to have similar data characteristics.

Cluster-based discretization:

  • For the continuous feature to be discretized, use a clustering algorithm (e.g., K-means, EM algorithm) to partition the samples into clusters or classes based on the distribution of that feature;

  • After finalizing the clusters, determine the cutpoints and the number of intervals.

Regression Models

Simple Linear Regression

In a regression model, the variable to be predicted is called the dependent variable (target variable), and the variables we choose to explain changes in the dependent variable are called independent variables (feature variables)

Simple linear regression model

Assumption: \( y = b + w_1 x + \varepsilon \), where \( b, w_1 \) are regression coefficients, \(\varepsilon\) is the random error term (noise), \(\varepsilon \sim N(0, \sigma^2)\), then the random variable \( y \sim N(b + w_1 x, \sigma^2) \).

Given sample set \( D = \{(x_1, y_1), ..., (x_n, y_n)\} \), the goal is to find a line \( y = b + w_1 x \) such that all sample points lie as close to it as possible.

\[ (\hat{b}, \hat{w}_1) = \arg \min_{(\hat{b}, \hat{w}_1)} \sum_{i=1}^n (y_i - b - w_1 x_i)^2 \]

Multiple Linear Regression Model (D features)

\[ \hat{y} = f(x) = \sum_{i=1}^{D} w_i x_i + b = w^T x + b \]

Loss function: Prediction error for a single sample

\[ \mathcal{L}(w, b) = \frac{1}{2}(\hat{y} - y)^2 = \frac{1}{2}(w^T x + b - y)^2 \]

Note: The factor \(\frac{1}{2}\) is only for computational convenience (cancels out the 2 during differentiation).

Cost function: Average of loss function over all training samples

\[ \mathcal{J}(w, b) = \frac{1}{2N} \sum_{i=1}^{N} (\hat{y}^{(i)} - y^{(i)})^2 = \frac{1}{2N} \sum_{i=1}^{N} (w^T x^{(i)} + b - y^{(i)})^2 \]

Gradient

\[ \nabla J(w, b) = \begin{bmatrix} \frac{\partial J}{\partial w} \\ \frac{\partial J}{\partial b} \end{bmatrix} = \begin{bmatrix} \frac{\partial J}{\partial w_1} & \frac{\partial J}{\partial w_2} & \cdots & \frac{\partial J}{\partial w_D} & \frac{\partial J}{\partial b} \end{bmatrix}^T \] \[ w = w - \alpha \frac{\partial J}{\partial w}, b = b - \alpha \frac{\partial J}{\partial b} \]

Gradient Descent

\[ \frac{\partial \mathcal{J}}{\partial w_j} = \frac{1}{N} \sum_{i=1}^{N} (w^T x^{(i)} + b - y^{(i)}) \cdot x_j^{(i)} \] \[ \frac{\partial \mathcal{J}}{\partial b} = \frac{1}{N} \sum_{i=1}^{N} (w^T x^{(i)} + b - y^{(i)}) \cdot 1 \] \[ \begin{bmatrix} w \\ b \end{bmatrix} - \alpha \begin{bmatrix} \frac{\partial \mathcal{J}}{\partial w} \\ \frac{\partial \mathcal{J}}{\partial b} \end{bmatrix} = \begin{bmatrix} w - \alpha \frac{\partial \mathcal{J}}{\partial w} \\ b - \alpha \frac{\partial \mathcal{J}}{\partial b} \end{bmatrix} \]

Least Squares Solution

Parameter vector w: Contains all weights \( w_1, \ldots, w_D \) and the bias term \( w_0 \). $w = (w_1, w_2, \ldots, w_D, w_0)^T$

Feature matrix X: Feature matrix of \( N \) samples, with one column of all 1's (for bias \( w_0 \))

Model prediction: $\hat{y} = Xw$

Objective: Find the optimal parameters w that minimize the Residual Sum of Squares (RSS)

Residual Sum of Squares (RSS) formula: Sum of squared errors between predicted values and true values y.

\[ RSS(w) = \|y - Xw\|_2^2 \]

Solution:

\[ \frac{\partial RSS}{\partial w} = -2X^T(y - Xw)=0 \] \[ \hat{w} = (X^T X)^{-1} X^T y \]

  • \( X^T X \) is called the Auto-correlation Matrix (after centering, it becomes the covariance matrix), denoted as \( R_x \in \mathbb{R}^{(D+1)\times (D+1)} \)

  • \( X^T y \in \mathbb{R}^{D+1} \) is called the Cross-correlation Vector

Limitations:

  • Insufficient samples: When the number of parameters exceeds the number of samples, the matrix \(X^TX\) is necessarily rank-deficient and has no inverse.

  • Multicollinearity: Even with sufficient samples, if strong collinearity exists among variables, \(X^TX\) becomes nearly singular, causing the diagonal entries of \((X^TX)^{-1}\) to be very large, increasing the variance of parameter estimates w and making them inaccurate.

Solutions: Regularization, Principal Component Regression, Partial Least Squares Regression

Advantages of Gradient Descent

  • GD is applicable to a wider range of models, especially nonlinear models.

  • Compared to direct solution methods (e.g., matrix inversion), GD is easier to implement.

  • For regression problems with high-dimensional features, GD is more efficient than LSM:

    • The matrix \( X X^T \) may not be invertible.

    • Matrix inversion has computational complexity \( O(D^3) \) (where D is feature dimension).

    • Each GD update has computational cost \( O(ND) \) (where N is number of samples, D is feature dimension).

Model Diagnostic Metrics for Linear Regression

\[ SS_{\text{tot}} = \sum_{i=1}^{n}(y_i - \bar{y})^2 \quad \text{Total sum of squares} \] \[ SS_{\text{res}} = \sum_{i=1}^{n}(y_i - \hat{y_i})^2 \quad \text{Residual sum of squares} \] \[ R^2(y, \hat{y}) = 1 - \frac{SS_{\text{res}}}{SS_{\text{tot}}} \]

$R^2 \approx 1$ indicates high model fit, $R^2 \approx 0$ indicates poor model fit.

Regularization in Linear Regression

Overfitting problem: Linear regression may overfit when the model has too many variables/features (ideal case: D+1 $<$ N)

Regularization: Simplifies the model by limiting the magnitude of the weight vector w.

  • Weight sparsity: Keeps few non-zero entries in w (simplifies structure), e.g., L1 regularization

  • Weight decay: Keeps the weight values in w small (increases smoothness), e.g., L2 regularization

  • Hyperparameter \(\lambda\): Balances "fitting error" (empirical risk) and "model complexity" (regularization term); larger \(\lambda\) leads to simpler model. \[ \sum_{i=1}^{n} (y_i - w^T x_i)^2 + \lambda \|w\|^q \]

    • \( q = 1 \) : LASSO

    • \( q = 2 \) : Ridge Regression

Ridge Regression

Objective function:

\[ \min_w \left\{ \sum_{i=1}^{n} (y_i - w^T x_i)^2 + \lambda \|w\|_2^2 \right\} \]

Differentiating w.r.t w and setting gradient to zero yields:

\[ \hat{w}^{\text{ridge}} = (X^T X + \lambda I)^{-1} X^T y \]

LASSO Regression

Objective function:

\[ \sum_{i=1}^{n} (y_i - w^T x_i)^2 + \lambda \|w\|_1 \]

The solution \(\hat{w}^{\text{LASSO}}\) has no closed-form expression. Common solution algorithms include coordinate descent, LARS algorithm, and ISTA algorithm.

ISTA algorithm: Objective function:

\[ F(w) = L(w) + R(w) \]

Where: \( L(w) \) is a differentiable smooth term (e.g., loss function), \( R(w) \) is a non-differentiable regularization term (e.g., L1 regularization)

Perform gradient descent only on the smooth term \( L(w) \), obtaining an intermediate result:

\[ v = w^{(k)} - \eta \nabla L(w^{(k)}) \]

Apply the proximal operator to the non-differentiable term \( R(w) \). For a convex function \( R(w) \) and step size \(\eta > 0\),

\[ w^{(k+1)} = \text{prox}_{\eta R}(v) \]

Here

\[ \text{prox}_{\eta R}(v) = \arg \min_w \left( R(w) + \frac{1}{2\eta} \|w - v\|_2^2 \right) \]

When \( R(w) = \lambda \|w\|_1 \), the analytic solution of \( \text{prox}_{\eta R}(v) \) is the soft-thresholding operator \( S_\tau(v) \):

\[ w^{(k+1)} = \text{prox}_{\eta (\lambda \|w\|_1)}(v) = S_{\eta \lambda}(v) \]

Soft-thresholding operator: Processes each component of vector \( v \) independently, with threshold \(\tau = \eta \lambda\).

\[ S_\tau (v_i) = \begin{cases} v_i - \tau & \text{if } v_i > \tau \\ v_i + \tau & \text{if } v_i < -\tau \\ 0 & \text{if } |v_i| \leq \tau \end{cases} \]

Principle of sparsity:

  • Any component whose absolute value is less than the threshold is set to 0.

  • Any component whose absolute value exceeds the threshold is shrunk towards 0 by \(\tau\).

Optimization Perspective

  • Constrained optimization problem (prior assumption) Assume \( \|w\|_p \leq C \) for parameter vector \( w \), leading to the constrained optimization problem: \[ \min_{w} \|y - Xw\|_2^2, \quad \text{s.t.} \quad \|w\|_2 \leq C \quad (\text{Ridge Regression}) \] \[ \min_{w} \|y - Xw\|_2^2, \quad \text{s.t.} \quad \|w\|_1 \leq C' \quad (\text{LASSO Regression}) \]
  • Lagrange multiplier method (unconstrained optimization) \[ \min_{w} \|y - Xw\|_2^2 + \lambda \|w\|_2^2 \quad (\text{Ridge Regression}) \] \[ \min_{w} \frac{1}{2} \|y - Xw\|_2^2 + \lambda \|w\|_1 \quad (\text{LASSO Regression}) \]

    Where the regularization coefficient \(\lambda > 0\) is a constant dependent on the constraint \(C\).

Limitations of Lasso

  • When features have linear correlation, Lasso tends to select only one of them, and the choice can be arbitrary. In practice, sometimes we prefer to select the entire group of features.

  • When samples are insufficient (feature count \(D > N\)), Lasso selects at most \(N\) features (non-zero coefficients), unable to handle high-dimensional data requiring more features.

Improvements to Lasso

I: Elastic Net

\[ \min_{w} \frac{1}{2} \|y - Xw\|_2^2 + \lambda_1 \|w\|_1 + \lambda_2 \|w\|_2^2 \]

Effect: Allows highly correlated features to have similar non-zero coefficients (for negatively correlated features, coefficient signs may differ).

II: Group Lasso

\[ \min_{w} \left\{ \frac{1}{2} \|y - Xw\|_2^2 + \lambda \sum_{j=1}^G \lambda_j \|w_j\|_2 \right\} \]

The total coefficient vector \( w \) is partitioned into G groups, where each \( w_j \) corresponds to coefficients of a feature group.

\[ w = (w_1, w_2, \ldots, w_G) \]

Suitable when features have natural group structures, e.g., a group of genes from the same biological process. Group Lasso performs group-level feature selection, achieving an effect between L1 and L2 regularization.

Nonlinear Regression

Spline curve: A piecewise polynomial curve formed by connecting several control nodes.

Spline regression: A regression model that fits functions using spline curves.

Linear spline regression: Fits a curve using multiple line segments, ensuring continuity at the connection points.

\[ f(x) = \underbrace{\beta_0 + \beta_1 x}_{\text{Base linear part}} + \underbrace{\sum_{j=1}^K w_j (x - a_j)_+}_{\text{Spline correction part}} \]

Truncated function $ (x - a_j)_+ = \max(0, x - a_j)$

  • Control nodes: $a_1 < a_2 < \cdots < a_K$, dividing the data range into multiple subintervals.

  • Parameter vector: $w = (\beta_0, \beta_1, w_1, w_2, \ldots, w_K)^T$, containing K+2 parameters to estimate.

  • Response vector: $y = (y_1, y_2, \ldots, y_n)^T$, containing observed values of n training samples.

  • Design matrix \[ G = \begin{pmatrix} 1 & x_1 & (x_1 - a_1)_+ & (x_1 - a_2)_+ & \cdots & (x_1 - a_K)_+ \\ 1 & x_2 & (x_2 - a_1)_+ & (x_2 - a_2)_+ & \cdots & (x_2 - a_K)_+ \\ \vdots & \vdots & \vdots & \ddots & \vdots & \vdots \\ 1 & x_n & (x_n - a_1)_+ & (x_n - a_2)_+ & \cdots & (x_n - a_K)_+ \end{pmatrix} \]

    • Base slope: Before the first knot, the model's slope is \(\beta_1\).

    • Slope change: Whenever \(x\) crosses a knot \(a_j\), the model's slope immediately increases by \(w_j\).

    • Cumulative effect: The final slope is the base slope \(\beta_1\) plus the sum of all activated spline term coefficients \(\sum w_j\).

Higher-order Spline Regression

Cubic spline curve

\[ y = \beta_0 + \beta_1 x + \beta_2 x^2 + \beta_3 x^3 + \sum_{j=1}^K w_j (x - a_j)_+^3 \]

  • \((x - a_j)_+^3\) is the cubic truncated power basis function, defined as \(\max(0, x - a_j)^3\).

  • Base part: The four terms \(\beta_0\) to \(\beta_3\) form a global cubic polynomial for fitting the data.

  • Spline part: Composed of \(w_j\) and their corresponding cubic truncation terms, responsible for local adjustments to the curve shape after knot \(a_j\).

Bias-Variance Decomposition

Total error = Bias\(^2\) + Variance + Irreducible error
Essential characteristic of overfitting: High variance, low bias

  • When training is insufficient, the learner's fitting ability is weak, bias dominates.

  • As training progresses, the learner's fitting ability strengthens, variance gradually dominates.

  • When training is sufficient, the learner's fitting ability is very strong, variance dominates.

Generalization error: Expected prediction error of model \(f\) at test point \(x\), denoted \(E(f; D)\)
For regression tasks: \(E(f; D) = \text{bias}^2(x) + \text{var}(x) + \epsilon^2\)

\[ E[f; D] = E_D [(f(x; D) - y_D)^2] \]

Add and subtract \(\bar{f}(x)\) inside the parentheses:

\[ E[f; D] = E_D [(f(x; D) - \bar{f}(x)) + (\bar{f}(x) - y_D))^2] \] \[ = E_D [(f(x; D) - \bar{f}(x))^2] + E_D [(\bar{f}(x) - y_D)^2] + 2E_D [(f(x; D) - \bar{f}(x))(\bar{f}(x) - y_D)] \]

The cross term \( 2E_D [(f(x; D) - \bar{f}(x))(\bar{f}(x) - y_D)] \) is zero because:

\[ E_D [(f(x; D) - \bar{f}(x))(\bar{f}(x) - y_D)] = (\bar{f}(x) - y_D) \cdot E_D [f(x; D) - \bar{f}(x)] \]

By definition \(\bar{f}(x) = E_D [f(x; D)]\), thus \( E_D [f(x; D) - \bar{f}(x)] = 0 \)
Decompose the second term \( E_D \left[ (\bar{f}(x) - y_D)^2 \right] \), introducing the true label \( y \) (add and subtract \( y \)):

\[ E_D \left[ (\bar{f}(x) - y_D)^2 \right] = E_D \left[ ((\bar{f}(x) - y) + (y - y_D))^2 \right] \] \[ = (\bar{f}(x) - y)^2 + E_D \left[ (y_D - y)^2 \right] + 2E_D \left[ (\bar{f}(x) - y)(y - y_D) \right] \]

Logistic Regression

Classification Problem (Supervised)

Given: Description of a sample \( x \) in sample space \( \mathcal{X} \) A fixed set of categories: \( C = \{ c_1, c_2, \cdots, c_m \} \)

Find: Category of \( x \): \( f(x) \in C \), where \( f(x) \) is a category function with domain \( \mathcal{X} \) and codomain \( C \).

Classification process:

Classifier training: Building a classification model using features and labels of training samples.

Prediction: Using the classification model to predict labels for unlabeled data.

Dataset partitioning into training set, validation set, and test set. The training and validation sets are used to train and select the classifier; the test set is used to evaluate classification performance.

  • True / False

    • True: Prediction = True Label

    • False: Prediction $\neq$ True Label

  • Positive / Negative

    • Positive: Prediction y = 1

    • Negative: Prediction y = 0

  • Accuracy: Proportion of correctly classified samples out of the total number of samples. \[ \text{Acc} = \frac{TP + TN}{TP + TN + FP + FN} \]

    When samples are imbalanced, accuracy is not a good metric for model evaluation.

  • Precision: Proportion of correctly predicted positive samples among all samples predicted as positive. \[ \text{Prec} = \frac{TP}{TP + FP} \]
  • Recall: Proportion of positive samples correctly predicted (recalled). \[ \text{Rec} = \frac{TP}{TP + FN} \] Trade-off between precision and recall \[ \hat{y} = \begin{cases} 1, & p_{\theta}(y = 1|x) > h \\ 0, & \text{otherwise} \end{cases} \]

    • Higher threshold \( h \) leads to higher precision but lower recall. Extreme case: threshold = 0.99.

    • Lower threshold \( h \) leads to lower precision but higher recall. Extreme case: threshold = 0.

  • F1 Score: Harmonic mean of precision and recall. \[ \text{F1} = \frac{2 \times \text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}} \]

ROC Curve (Receiver Operating Characteristic Curve)

  • Vertical axis: TPR (True Positive Rate) \[ \text{TPR} = \frac{TP}{TP + FN} = \frac{TP}{\text{Total actual positives}} \]
  • Horizontal axis: FPR (False Positive Rate) \[ \text{FPR} = \frac{FP}{FP + TN} = \frac{FP}{\text{Total actual negatives}} \]

AUC (Area Under the ROC Curve): The probability that the model ranks a randomly chosen positive sample higher than a randomly chosen negative sample. Measures the global ranking quality of the model.

\[ AUC = P(\text{Score}_P > \text{Score}_N) \] \[ AUC = \frac{1}{M \cdot N} \sum_{i=1}^{M} \sum_{j=1}^{N} I(\text{Score}_{P_i} > \text{Score}_{N_j}) \]

Advantages of ROC / AUC: The model's inherent ability to separate positive and negative examples across all possible classification thresholds. Because the denominators of TPR and FPR only depend on the respective actual class counts, it effectively ignores class distribution bias in the data, providing a fair and objective view of model performance.

  • Unaffected by class proportion: Allows fair comparison of model performance across datasets with vastly different positive/negative sample ratios.

  • Model-agnostic: Can directly compare any type of classification model.

  • Independent of business context: Enables comparison of models' intrinsic discrimination ability solely based on the curve/AUC across different domains (e.g., medical diagnosis vs. fraud detection).

Logistic Regression (Classification)

Cross-entropy loss: $H(p,q) = -\sum_x p(x) \log q(x)$ Self-information: $I(x) = -\log_2 P(x)$

Shannon entropy (Surprise): Entropy \( H(P) \) is the expectation of self-information, measuring the uncertainty of distribution \( P \) itself. (For a discrete distribution with N possible outcomes, entropy is maximized when uniform.)

\[ H(P) = E_P[I(x)] = -\sum_x P(x) \log_2 P(x) \]

KL Divergence (Relative entropy): Measures the distance between two distributions (always >= 0, zero when equal).

\[ D_{\text{KL}}(p \| q) = \sum_x p(x) \log \frac{p(x)}{q(x)} \] \[ H(p,q) = H(p) + D_{\text{KL}}(p \| q) \]

Loss function: $\mathcal{L}(y,x,p_\theta) = -\delta(y = 1) \log p_\theta (y = 1|x) - \delta(y = 0) \log p_\theta (y = 0|x)$

$$ = -y \log p_\theta (y = 1|x) - (1-y) \log (1-p_\theta (y = 1|x)) = -y \log \sigma(\theta^T x) - (1-y) \log (1-\sigma(\theta^T x))$$ $$\frac{\partial \mathcal{L}(y,x,p_\theta)}{\partial \theta} = (\sigma(\theta^T x) - y)x$$ \[ \theta \leftarrow \theta + \eta(y - \sigma(\theta^T x))x \]

Parameter estimation (with \(y_i \in \{-1,1\}\)):

\[ p(y_i | x_i) = \sigma(y_i w^T x_i) \] \[ \text{NLL}(w) = -\sum_{i=1}^n \log \sigma(y_i w^T x_i) \] \[ \nabla \text{NLL}(w) = -\sum_{i=1}^n (1-\sigma(y_i w^T x_i)) y_i x_i \] \[ w \leftarrow w - \eta \nabla \text{NLL}(w) \]

Multi-class cross-entropy loss

\[ \mathcal{L}(y,x,p_\theta) = -\sum_k \delta(y = c_k) \log p_\theta(y = c_k|x) \]

Softmax: \( m \) denotes the total number of classes: $p_\theta(y = c_j|x) = \frac{e^{\theta_j^T x}}{\sum_{k=1}^m e^{\theta_k^T x}} \quad \text{for } j = 1, \cdots, m$

\[ \frac{\partial \log p_\theta(y = c_j|x)}{\partial \theta_j} = \frac{\partial}{\partial \theta_j} \log \frac{e^{\theta_j^T x}}{\sum_{k=1}^m e^{\theta_k^T x}} \] \[ = x - \frac{\partial}{\partial \theta_j} \log \sum_{k=1}^m e^{\theta_k^T x} \] \[ = x - \frac{e^{\theta_j^T x}}{\sum_{k=1}^m e^{\theta_k^T x}} x = \left(1 - p_\theta(y = c_j|x)\right)x \]

K-Nearest Neighbors (Non-parametric statistical method for classification and regression)

Distance metrics: Euclidean distance (most common); Minkowski distance; Cosine similarity
Feature normalization/weighting: Z-Score standardization
Choice of K (K should ideally be odd): K = 1 is most common and often effective, but sensitive to "noise".

  • Small K: Sensitive to "noise", non-smooth decision boundary (may lead to overfitting).

  • Large K: Generally produces smoother decision boundaries, can reduce the impact of noisy samples (note: overly smooth decision boundaries may lead to underfitting).

Dataset partitioning: K-fold cross-validation

  • Randomly and evenly partition the original dataset into K subsets of similar size.

  • Each time, use K-1 subsets as the training set and the remaining 1 as the validation set.

  • Repeat K times, each time using a different subset as the validation set.

  • Finally, aggregate the K results to obtain a performance estimate (take the average).

Curse of Dimensionality: As dimensionality increases, the number of samples needed to cover a fixed proportion of the sample space grows rapidly. Curse of dimensionality = Sparsity + Distance breakdown

Complexity Reduction with k-d Trees (k denotes dimensionality, distinct from K in KNN)

  1. Establish the root node.

  2. Select the feature with the largest variance as the splitting feature (variance of all data samples in the current node along a particular feature dimension), or for a node at depth j, choose \(x^{(j)}\) as the splitting axis \(l = j \ (\text{mod} \ k) + 1\) (avoiding variance computation).

  3. Choose the median of that feature as the split point.

  4. Pass samples with feature values less than the median to the left subtree, and those greater than the median to the right subtree.

  5. Recursively execute steps 2-4 until all data is assigned to nodes in the KD Tree.

KD Tree complexity: Logarithmic time complexity: O(log n); without KD Tree, linear time complexity: O(n)
Limitations of KD Trees:

  • High-dimensional space problem: kd trees are not suitable for fast nearest neighbor queries in high-dimensional spaces (k $>$ 20). Consider alternatives like Ball-Trees.

  • Inefficient for dynamic data updates: kd trees are not suitable for data streams or frequently updated databases.

Decision Trees (Supervised Learning)

There are two types of nodes: internal nodes and leaf nodes. Internal nodes represent a feature or attribute, leaf nodes represent a class.

ID3

Goal: A decision tree with low prediction error on the training set while having good generalization ability.

Loss function: Theoretically, it is a regularized maximum likelihood function. However, finding the globally optimal decision tree based on a loss function is an NP problem. In practice, decision tree learning uses heuristic methods (greedy), i.e., local optimum.

Feature Selection (Features with classification ability for the training data)

Information Gain / Mutual Information: The difference between the empirical entropy H(D) of dataset D and the empirical conditional entropy H(D|A) of D given feature A, \(g(D,A)=H(D)-H(D|A)\)
Conditional entropy $H(X|Y)$: Measures the remaining uncertainty in random variable X after receiving information from random variable Y.

\[ H(X|Y)=\sum_{v\in \text{values}(Y)}P(Y=v)H(X|Y=v) \] \[ I(X,Y)=H(X)-H(X|Y)=H(Y)-H(Y|X)=H(X)+H(Y)-H(X,Y) \]

  • If X and Y are completely independent: Knowing Y provides no information about X, so \(H(X|Y)=H(X)\), then \(I(X;Y)=H(X)-H(X)=0\). Mutual information being zero proves independence.

  • If X and Y are perfectly correlated: Knowing Y completely determines X, so \(H(X|Y)=0\), then \(I(X;Y)=H(X)\). Mutual information equals the total uncertainty of X.

Proof:

\[ I(X,Y)=H(X)-H(X|Y) \] \[ =-\sum_{v}P(X=v)\log P(X=v)+\sum_{u}P(Y=u)\sum_{v}P(X=v|Y=u)\log P(X=v|Y=u) \] \[ =-\sum_{v}P(X=v)\log P(X=v)+\sum_{u}\sum_{v}P(X=v,Y=u)\log P(X=v|Y=u) \] \[ =-\sum_{v}P(X=v)\log P(X=v)+\sum_{u}\sum_{v}P(X=v,Y=u)[\log P(X=v,Y=u)-\log P(Y=u)] \] \[ =-\sum_{v}P(X=v)\log P(X=v)-\sum_{u}P(Y=u)\log P(Y=u)+\sum_{u,v}P(X=v,Y=u)\log P(X=v,Y=u) \] \[ =H(X)+H(Y)-H(X,Y) \]

Computation

Let the training dataset be D, \(|D|\) denotes its sample size (number of samples). Suppose there are K classes \(C_k\), \(k=1,2,...,K\). \(|C_k|\) is the number of samples belonging to class \(C_k\), \(\sum_{k=1}^{K}|C_k|=|D|\).

Suppose feature A has n distinct values \(\{a_1,a_2,...,a_n\}\). Partition D into n subsets \(D_1,D_2,...,D_n\) according to the values of feature A. \(|D_i|\) is the number of samples in \(D_i\), \(\sum_{i=1}^{n}|D_i|=|D|\).

Let \(D_{ik}\) be the set of samples in subset \(D_i\) belonging to class \(C_k\), and \(|D_{ik}|\) be its size.

\[ H(D) = -\sum_{k=1}^{K} \frac{|C_k|}{|D|} \log_2 \frac{|C_k|}{|D|} \] \[ H(D|A) = \sum_{i=1}^{n} \frac{|D_i|}{|D|} H(D_i) = -\sum_{i=1}^{n} \frac{|D_i|}{|D|} \sum_{k=1}^{K} \frac{|D_{ik}|}{|D_i|} \log_2 \frac{|D_{ik}|}{|D_i|} \]

C4.5

Information gain ratio:

\[ g_R(D, A) = \frac{g(D, A)}{H_A(D)} \]

Decision tree generation

  1. Determine the classification attribute.

  2. Create a node N for the current data table.

  3. If all data in the database belong to the same class, N is a leaf, label the leaf with that class.

  4. If there are no other attributes to consider in the data table, N is also a leaf, label it with the majority class (majority vote).

  5. Otherwise, select the best attribute based on information gain as the test attribute for node N.

  6. After the node attribute is selected, for each value of that attribute: generate a branch from N, collect the data related to that branch from the data table to form the data table for the branch node, delete the node attribute column from that table. If the branch node's data table is non-empty, apply the above algorithm recursively to build the subtree from that node.

ID3 vs C4.5

  • For discrete categorical data: one feature value or category forms one branch.

  • ID3 algorithm tends to favor features with many values (especially when the number of distinct values is comparable to the sample size): ID3 considers "pure" child nodes to be good, and features with many values can easily "split" into child nodes containing very few samples that are extremely "pure", achieving the highest gain score. This is an overfitting tendency.

  • C4.5: Tends to favor features with uneven value distribution. By introducing the "gain ratio" to penalize features with many values, it improves ID3's bias (numerator (information gain) → often very large; denominator (SplitInfo) → also very large (due to many values possibly uniformly distributed); ratio (gain ratio) → may become smaller).

Decision Tree Pruning

Prediction error: The sum of empirical entropies of all leaf nodes. The number of leaf nodes in tree T is \(|T|\). t is a leaf node of tree T. \(N_t\) is the number of sample points at that leaf node, of which \(N_{tk}\) belong to class k (\(k=1,2,...,K\)). \(H_t(T)\) is the empirical entropy of the leaf node.

\[ H_t(T) = -\sum_k \frac{N_{tk}}{N_t} \log \frac{N_{tk}}{N_t} \]

Loss function for decision tree learning

\[ C_\alpha(T) = \sum_{t=1}^{|T|} N_t H_t(T) + \alpha |T| \]

Objective: Find the tree that minimizes the total loss.

  1. Calculate the empirical entropy for each node.

  2. Recursively prune upwards from leaf nodes. Let the overall trees before and after pruning a set of leaf nodes back to their parent node be \(T_B\) and \(T_A\) respectively, with corresponding loss functions \(C_\alpha(T_B)\) and \(C_\alpha(T_A)\). If $C_\alpha(T_A) < C_\alpha(T_B)$, perform the pruning, i.e., turn the parent node into a new leaf node.

  3. Return to step (2) until no further pruning is possible, obtaining the subtree \(T_\alpha\) that minimizes the loss function.

  • Pre-pruning

    1. Define a height; stop tree growth when the decision tree reaches that height.

    2. If instances at a node have identical feature vectors, even if they do not belong to the same class, stop tree growth. This method is effective for handling data conflicts.

    3. Define a threshold; stop tree growth when the number of instances at a node falls below that threshold.

    4. Define a threshold; decide whether to stop tree growth by calculating the performance gain from each expansion and comparing it to the threshold.

    Pre-pruning methods are relatively simple, highly efficient, and do not require generating the entire decision tree, making them suitable for large-scale problems.

  • Post-pruning (Using criteria to determine the final correct tree size)

    1. Use a training set and a validation set to evaluate the effectiveness of pruning methods on nodes.

    2. Train using the entire training set but use statistical tests to estimate whether pruning a specific node would improve performance on data outside the training set. Test whether further expanding a node improves classification performance on the overall data or merely on the current training data.

CART (Binary Tree)

Regression tree: Uses least squares deviation minimization criterion.

  1. Decision tree generation: Generate a decision tree based on the training dataset, making it as large as possible.

  2. Decision tree pruning: Prune the generated tree using a validation dataset and select the optimal subtree, using loss function minimization as the pruning criterion.

Support Vector Machine (Optimization algorithm solving convex quadratic programming)

Basic model: A linear classifier defined on the feature space that maximizes the margin. Learning strategy: Margin maximization.

Linear Classifier

Empirical Risk Minimization: (Logistic Regression)

Training objective: Minimize cross-entropy loss on the training set.

Parametric model: Influenced by every sample in the dataset.

Logit value in Logistic Regression: $s(x) = \theta_0 + \theta_1 x_1 + \theta_2 x_2, p_\theta(y = 1|x) = \frac{1}{1 + e^{-s(x)}}$

  • The Logit value reflects the distance from the sample to the decision boundary $d = \frac{\left| \theta_1 x_1 + \theta_2 x_2 + \theta_0 \right|}{\sqrt{\theta_1^2 + \theta_2^2}}$.

  • The sign of the Logit value determines the class.

  • The absolute value of the Logit value represents classification confidence.

  • Higher Logit values indicate samples farther from the decision boundary, classified with more confidence.

Structural Risk Minimization

Training objective: Find the maximum margin boundary among many decision boundaries (separating hyperplanes).

Non-parametric model, influenced only by a few core data points.
Functional margin: Scaling Problem (multiplying by \(y^{(i)}\) ensures the margin is always positive).

\[ \hat{\gamma}^{(i)} = y^{(i)} (w^\top x^{(i)} + b) \]

The separating hyperplane does not change with the magnitude of \((w, b)\).

\[ g(w^\top x + b) = g(2w^\top x + 2b) \]

Functional margin of the training dataset

\[ \hat{\gamma} = \min_{i=1, \cdots, N} \hat{\gamma}^{(i)} \]

Geometric margin: To overcome the scaling problem of the functional margin, normalize the normal vector \( w \).

  • For a given training dataset and hyperplane \((w, b)\) \[ \gamma_i = y_i \left( \frac{w}{\|w\|} \cdot x_i + \frac{b}{\|w\|} \right) \]

  • Geometric margin of the training dataset \(\gamma = \min_{i=1, \cdots , N} \gamma_i\)

Margin Maximization

Original optimization problem for maximizing the geometric margin

\[ \begin{aligned} &\max_{w,b} \gamma \\ &\text{s.t. } y_i \left( \frac{w}{\|w\|} \cdot x_i + \frac{b}{\|w\|} \right) \geq \gamma, \quad i = 1, 2, \cdots , N \end{aligned} \]

Optimization problem after expressing the geometric margin in terms of functional margin

\[ \begin{aligned} &\max_{w,b} \frac{\hat{\gamma}}{\|w\|} \\ &\text{s.t. } y_i (w \cdot x_i + b) \geq \hat{\gamma}, \quad i = 1, 2, \cdots , N \end{aligned} \]

Conveniently set the minimum functional margin \(\hat{\gamma} = 1\).

Optimization problem for learning a linearly separable Support Vector Machine

\[ \begin{aligned} &\min_{w,b} \frac{1}{2} \|w\|^2 \\ &\text{s.t. } y_i (w \cdot x_i + b) - 1 \geq 0, \quad i = 1, 2, \cdots , N \end{aligned} \]

Objective function: For computational convenience (convex optimization).

Constraints: Ensure all sample points are correctly classified and lie outside the maximum margin boundaries (functional margin at least 1).

Lagrange Duality Problem

\[ \begin{aligned} &\min_{w} f(w)\\ &\text{s.t. } g_i(w) \leq 0, i = 1, \ldots, k \\ &\quad h_i(w) = 0, i = 1, \ldots, l \end{aligned} \]

Definition of Lagrange function:

\[ \mathcal{L}(w, \alpha, \beta) = f(w) + \sum_{i=1}^{k} \alpha_i g_i(w) + \sum_{i=1}^{l} \beta_i h_i(w) \]

Primal problem: \(\theta_P(w) = \max_{\alpha \geq 0, \beta} \mathcal{L}(w, \alpha, \beta)\)

Here \(\theta_P(w)\) is for a fixed \(w\), maximizing the Lagrange function over the multiplier space.

  • Case 1: \(w\) violates constraints.

    • Violates inequality constraint (some \(g_i(w) > 0\)), take the corresponding \(\alpha_i \to +\infty\): \[ \mathcal{L}(w, \alpha, \beta) = f(w) + \alpha_i g_i(w) + \text{other terms} \]

      Because \(g_i(w) > 0, \alpha_i > 0\), so \(\alpha_i g_i(w) \to +\infty \Rightarrow \max \mathcal{L} \to +\infty \Rightarrow \theta_P(w) = +\infty\).

    • Violates equality constraint (some \(h_i(w) \neq 0\)) We can take the corresponding \(\beta_i \to \pm \infty\) (with the sign matching \(h_i(w)\)): \[ \beta_i h_i(w) \to +\infty \Rightarrow \theta_P(w) = +\infty \]

      Conclusion: As long as \(w\) violates any constraint, \(\theta_P(w) = +\infty\).

  • Case 2: w satisfies all constraints.

    • For inequality constraints: \( g_i(w) \leq 0 \), and \( \alpha_i \geq 0 \) So \( \alpha_i g_i(w) \leq 0 \).

    • For equality constraints: \( h_i(w) = 0 \), so \( \beta_i h_i(w) = 0 \).

    To maximize \( \mathcal{L}(w, \alpha, \beta) \):

    \[ \mathcal{L}(w, \alpha, \beta) = f(w) + \sum_{i=1}^{k} \alpha_i g_i(w) + \sum_{i=1}^{l} \beta_i h_i(w) \]

    The maximum possible value is \( f(w) \), achieved if and only if all \( \alpha_i g_i(w) = 0 \).

    Because \( \alpha_i \geq 0 \), \( g_i(w) \leq 0 \), to have \( \alpha_i g_i(w) = 0 \), we need: \( g_i(w) = 0 \) or \( \alpha_i = 0 \). Thus:

    \[ \theta_p(w) = \max_{\alpha \geq 0, \beta} \mathcal{L}(w, \alpha, \beta) = f(w) \]

The original constrained optimization problem is equivalent to the unconstrained optimization of the primal function:

\[ \min_{w} \theta_p(w) = \min_{w} \max_{\alpha, \beta: \alpha_i \geq 0} \mathcal{L}(w, \alpha, \beta) \]

Dual problem

\[ \max_{\alpha, \beta: \alpha_i \geq 0} \theta_d(\alpha, \beta) = \max_{\alpha, \beta: \alpha_i \geq 0} \min_{w} \mathcal{L}(w, \alpha, \beta) \]

That is, swap the order of \(\min\) and \(\max\) in the primal problem.

\[ \min_{w} \theta_p(w) = \min_{w} \max_{\alpha, \beta: \alpha_i \geq 0} \mathcal{L}(w, \alpha, \beta) \]

Define the solution to the dual problem as

\[ d^* = \max_{\alpha, \beta: \alpha_i \geq 0} \min_{w} \mathcal{L}(w, \alpha, \beta) \]

It can be shown that \( d^* = \max_{\alpha, \beta: \alpha_i \geq 0} \min_{w} \mathcal{L}(w, \alpha, \beta) \leq \min_{w} \max_{\alpha, \beta: \alpha_i \geq 0} \mathcal{L}(w, \alpha, \beta) = p^* \)

\[ \theta_d(\alpha, \beta) \leq \theta_p(w) \]

Proof:

\[ \min_{w} \mathcal{L}(w, \alpha, \beta) \leq \mathcal{L}(w, \alpha, \beta), \forall w, \alpha \geq 0, \beta \] \[ \Rightarrow \max_{\alpha, \beta: \alpha \geq 0} \min_{w} \mathcal{L}(w, \alpha, \beta) \leq \max_{\alpha, \beta: \alpha \geq 0} \mathcal{L}(w, \alpha, \beta), \forall w \] \[ \Rightarrow \max_{\alpha, \beta: \alpha \geq 0} \min_{w} \mathcal{L}(w, \alpha, \beta) \leq \min_{w} \max_{\alpha, \beta: \alpha \geq 0} \mathcal{L}(w, \alpha, \beta) \]

Under certain conditions (e.g., Slater's condition for convex problems), strong duality holds:

\[ d^* = p^* \]

KKT Conditions

Assume \( f \) and \( g_i \) are convex functions, \( h_i \) are affine functions, and the feasible region is strictly feasible (Slater's condition). Then there exist \((w^*, \alpha^*, \beta^*) \) satisfying:

  • \( w^* \) is a solution to the primal problem.

  • \( \alpha^*, \beta^* \) are solutions to the dual problem.

  • The optimal values are equal \( p^* = d^* = \mathcal{L}(w^*, \alpha^*, \beta^*) \).

Furthermore, \( (w^*, \alpha^*, \beta^*) \) satisfy the KKT conditions:

  1. Stationarity: Gradient of the Lagrangian w.r.t primal variables \( w \) is zero. $\frac{\partial}{\partial w_i} \mathcal{L}(w^*, \alpha^*, \beta^*) = 0, i = 1, ..., n$

  2. Primal feasibility: $\frac{\partial}{\partial \beta_i} \mathcal{L}(w^*, \alpha^*, \beta^*) = 0, i = 1, ..., l \quad \text{and} \quad g_i(w^*) \leq 0, i = 1, ..., k$

  3. Dual feasibility: $\alpha_i^* \geq 0, i = 1, ..., k$

  4. Complementary slackness: $\alpha_i^* g_i(w^*) = 0, i = 1, ..., k$

Objective function of Support Vector Machine: Finding the optimal margin classifier

\[ \begin{aligned} &\min_{w,b} \frac{1}{2} \|w\|^2 \\ &\text{s.t. } -y^{(i)}(w^T x^{(i)} + b) + 1 \leq 0, \quad i = 1, ..., m \end{aligned} \]

Lagrange function (Note: No equality constraints in standard SVM)

\[ L(w, b, \alpha) = \frac{1}{2} \|w\|^2 - \sum_{i=1}^m \alpha_i [y^{(i)}(w^T x^{(i)} + b) - 1] \]

Partial derivatives with respect to the two parameters

\[ \frac{\partial}{\partial w} L(w, b, \alpha) = w - \sum_{i=1}^m \alpha_i y^{(i)} x^{(i)} = 0 \Rightarrow w = \sum_{i=1}^m \alpha_i y^{(i)} x^{(i)} \] \[ \frac{\partial}{\partial b} L(w, b, \alpha) = \sum_{i=1}^m \alpha_i y^{(i)} = 0 \]

Rewrite the Lagrange function (substituting the expression for w and using the constraint on the sum of $alpha_i y_i$)

\[ \begin{aligned} L(w, b, \alpha) &= \frac{1}{2} \left( \sum_{i=1}^m \alpha_i y^{(i)} x^{(i)} \right)^T \left( \sum_{j=1}^m \alpha_j y^{(j)} x^{(j)} \right) - \sum_{i=1}^m \alpha_i y^{(i)} \left( \sum_{j=1}^m \alpha_j y^{(j)} x^{(j)^T} \cdot x^{(i)} + b \right) + \sum_{i=1}^m \alpha_i \\ &= \sum_{i=1}^m \alpha_i - \frac{1}{2} \sum_{i,j=1}^m y^{(i)} y^{(j)} \alpha_i \alpha_j x^{(i)^T} x^{(j)} - b \sum_{i=1}^m \alpha_i y^{(i)} \\ &= \sum_{i=1}^m \alpha_i - \frac{1}{2} \sum_{i,j=1}^m y^{(i)} y^{(j)} \alpha_i \alpha_j x^{(i)^T} x^{(j)} \quad (\text{since } \sum_i \alpha_i y^{(i)} = 0) \end{aligned} \]

Lagrange dual problem

\[ \max_{\alpha \geq 0} \theta_D(\alpha) = \max_{\alpha \geq 0} \min_{w,b} L(w, b, \alpha) \] \[ \max_{\alpha} W(\alpha) = \sum_{i=1}^m \alpha_i - \frac{1}{2} \sum_{i,j=1}^m y^{(i)} y^{(j)} \alpha_i \alpha_j (x^{(i)})^T x^{(j)} \]

s.t.

\[ \alpha_i \geq 0, \quad i = 1, ..., m \] \[ \sum_{i=1}^m \alpha_i y^{(i)} = 0 \]

Soft Margin SVM

Introduce slack variables

\[ \min_{w,b,\xi} \frac{1}{2} \| w \|^2 + C \sum_{i=1}^{m} \xi_i \]

s.t.

\[ y^{(i)} (w^T x^{(i)} + b) \geq 1 - \xi_i, \quad i = 1,...,m \] \[ \xi_i \geq 0, \quad i = 1,...,m \]

Lagrange function (including multipliers for slack variables)

\[ L(w,b,\xi,\alpha,r) = \frac{1}{2} w^T w + C \sum_{i=1}^{m} \xi_i - \sum_{i=1}^{m} \alpha_i [y^{(i)} (x^{(i)^T} w + b) - 1 + \xi_i] - \sum_{i=1}^{m} r_i \xi_i \]

Dual problem (after minimizing over w, b, xi)

\[ \max_{\alpha} W(\alpha) = \sum_{i=1}^{m} \alpha_i - \frac{1}{2} \sum_{i,j=1}^{m} y^{(i)} y^{(j)} \alpha_i \alpha_j (x^{(i)})^T x^{(j)} \]

s.t.

\[ 0 \leq \alpha_i \leq C, \quad i = 1,...,m \] \[ \sum_{i=1}^{m} \alpha_i y^{(i)} = 0 \]

Equivalent unconstrained form (primal):

\[ \frac{1}{2} \| w \|^2 + C \sum_{i=1}^{m} \max \left( 0, 1 - y_i (w^T x_i + b) \right) \]

Kernel Methods

Feature mapping function Basic SVM focuses on inner product computations.

\[ W(\alpha) = \sum_{i=1}^{m} \alpha_i - \frac{1}{2} \sum_{i,j=1}^{m} y^{(i)} y^{(j)} \alpha_i \alpha_j x^{(i)^T} x^{(j)} \]

Define a feature mapping function \(\phi(x)\). The kernel function is \(K(x^{(i)}, x^{(j)}) = \phi(x^{(i)})^T \phi(x^{(j)})\). The dual becomes:

\[ W(\alpha) = \sum_{i=1}^{m} \alpha_i - \frac{1}{2} \sum_{i,j=1}^{m} y^{(i)} y^{(j)} \alpha_i \alpha_j K(x^{(i)}, x^{(j)}) \]

Kernel Function

\[ K(x^{(i)}, x^{(j)}) = \phi(x^{(i)})^{T} \phi(x^{(j)}) \]

Kernel Trick: In many cases, we can directly define \(K(x^{(i)}, x^{(j)})\) without needing to explicitly define \(\phi(x^{(i)})\). For example, suppose \(x^{(i)}, x^{(j)} \in \mathbb{R}^n\), then \(K(x^{(i)}, x^{(j)}) = \left( x^{(i)^T} x^{(j)} \right)^2\) corresponds to a mapping \(\phi\) into the space of degree-2 monomials. For a finite set of examples \(\{x^{(1)}, ..., x^{(m)}\}\), the corresponding kernel matrix \(K\) is defined as \(\{k_{i,j}\}_{i,j=1,...,m}\).

  • The kernel matrix \(K\) must be symmetric because \[ k_{i,j} = K(x^{(i)}, x^{(j)}) = \phi(x^{(i)})^{T} \phi(x^{(j)}) = \phi(x^{(j)})^{T} \phi(x^{(i)}) = K(x^{(j)}, x^{(i)}) = k_{j,i} \]

Mercer's Theorem: Given \(K: \mathbb{R}^n \times \mathbb{R}^n \mapsto \mathbb{R}\), if \(K\) is a valid (Mercer) kernel, then for any finite set \(\{x^{(1)}, ..., x^{(m)}\}\), the corresponding kernel matrix is symmetric positive semidefinite.

Ensemble Learning

Why can it improve performance?

\[ \text{Error}_H = \sum_{k \leq \frac{T}{2}} C_T^k p^{T-k}(1-p)^k \]

where p is the probability of a base classifier being correct, assuming independence. (Here, $k \le T/2$ means the number of correctly classified base classifiers is at most half.)

  • Bagging Randomly sample samples or features with replacement, learn multiple independent models, then average the predictions of all models. Primarily reduces variance. Representative algorithm: Random Forest.

  • Boosting Train multiple models sequentially, where later models are based on the errors of previous models. Primarily reduces bias. Representative algorithm: AdaBoost.

  • Stacking Use the predictions of multiple base models as input features and the sample labels as the target, then train a high-level ensemble model.

    • Advantages:

      1. Can use different types of base learners to capture different patterns in the data.

      2. Theoretically can combine the strengths of multiple models to achieve stronger predictive power.

    • Disadvantages:

      1. Training process is complex, requiring training multiple models, and the combination method needs careful design.

      2. More complex than other ensemble methods and prone to overfitting.

Bagging and Random Forests

Obtain bootstrap samples \(D_t\) of the same size as the original dataset D by sampling with replacement.

Given a dataset D with N samples, for each iteration \(t = 1, 2, ..., T\), create T base classifiers \(h_t\).

On average, 36.8% of the samples in D will not appear in a given bootstrap sample \(D_t\) (as N is large enough):

\[ P(x \text{ is not selected}) = \left(1 - \frac{1}{N}\right)^{N} \approx e^{-1} = 36.8\% \]

This means each training set contains approximately 63.2% distinct samples from the original dataset. Each \(D_t\) is used to learn a classifier \(h_t\).

  • For classification problems, each classifier returns a class prediction, combined using majority voting.

  • For regression problems, predictions are combined using averaging.

Advantages of Bagging:

  • Particularly suitable for improving the performance of base models that have high variance but low bias (e.g., decision trees, neural networks).

  • Individual models are unstable: small changes in the training data can cause significant changes in classifier performance.

  • Bagging aggregates votes/results, enhancing stability and accuracy.

  • Easy to parallelize: the sampling and training of multiple base models are independent, allowing parallel computation.

Random Forest (An improved version of Bagging for Decision Trees)

Characteristics:

  • Bootstrap sampling of samples (with replacement).

  • Random sampling of features.

  • Most commonly uses decision trees as the base classifier.

Limitations of Decision Trees:

  • Local optimum problem: Decision trees use a greedy algorithm, selecting the feature that provides the highest information gain at each split, which can lead to local optima.

  • Classification boundary problem: A single decision tree determines classification boundaries using logical conditions involving single features, resulting in decision boundaries that are axis-aligned lines/planes. This limits the expressive power of decision trees for classification boundaries, leading to poorer generalization.

Algorithmic Principles of Random Forests To ensure independence among the trees, typically two or three levels of randomness are introduced:

  • Randomly sample samples with replacement (doesn't see all data).

  • Randomly select m features (doesn't see all features).

  • (Sometimes) Randomly select a feature split point without evaluating all possible splits (non-perfect splitting criterion).

How many features to select (m)?

  • A large m value can improve the accuracy of individual decision trees but increases correlation among trees.

  • Method 1: Cross-validation to select the optimal m.

  • Method 2: Empirical setting. Often choose \(m = \log_2 d + 1\), where d is the feature dimension.

Performance Evaluation for Random Forest Classification margin: The proportion of decision trees correctly classifying a sample minus the proportion of trees predicting the most frequent incorrect class.

Assume a sample is \((X,Y)\), where Y is the true class. Random Forest is a collection of classifiers \(h_1(x), h_2(x), ..., h_k(x)\).

For sample X, define its classification margin \(mg(X,Y)\) as:

\[ mg(X,Y) = \text{av}_k(I(h_k(X) = Y)) - \max_{j\neq Y} \text{av}_k(I(h_k(X) = j)) \]

  • \(\text{av}_k(I(h_k(X) = Y))\): The proportion of trees that predict the correct class Y.

  • \(\max_{j\neq Y} \text{av}_k(I(h_k(X) = j))\): The highest proportion of trees predicting any incorrect class j.

Out-of-Bag (OOB) samples: For each tree, the samples that were NOT selected in its bootstrap training set.

Out-Of-Bag Error (OOBE): The prediction error rate of the Random Forest on the OOB samples. Random Forest for Feature Selection

  • Calculate the average information gain of features.

  • Calculate the impact of each feature on model accuracy (e.g., by permuting feature values and measuring the increase in OOB error).

Advantages:

  1. High Accuracy &amp; Robustness

    • Resists overfitting: Bagging averages results from multiple decision trees, effectively reducing model variance and improving generalization.

    • Noise tolerant: Random sampling of samples and features provides resilience to outliers and noise in the data.

  2. Ease of Use &amp; Low Preprocessing Requirements

    • No feature scaling needed: Tree-based algorithms do not require normalization or standardization.

    • Broad applicability: Can handle both classification and regression tasks and performs well with high-dimensional features.

  3. Interpretability &amp; Feature Insights

    • Feature importance evaluation: Can output feature importance scores, aiding feature selection and business understanding.

    • Built-in validation: Uses Out-Of-Bag (OOB) error for unbiased estimation, eliminating the need for a separate validation set.

  4. Efficient Parallelism

    • The construction of each tree is independent, allowing parallelized training using multi-core CPUs, significantly reducing training time.

Disadvantages:

  • Tends to overfit on small, noisy datasets or low-dimensional datasets.

  • Compared to a single decision tree: Prediction requires traversing all trees and aggregating results, which is slower.

  • Compared to a single decision tree: The complex decision logic of many intertwined trees is harder to visualize intuitively, offering lower interpretability.

  • Large model size: A model with hundreds of deep trees consumes significant memory.

Assume the variance of a single decision tree is \(\sigma^2(x)\), and the correlation between any two trees is \(\rho(x)\). If the Random Forest consists of T trees, the variance \(\text{var}(x)\) of the ensemble is:

\[ \text{var}(x) = \rho(x) \sigma^2(x) + \frac{1 - \rho(x)}{T} \sigma^2(x) \]

Derivation (assuming all trees have identical variance \(\sigma^2(x)\) and pairwise correlation \(\rho(x)\)):

\[ \text{Var}(\bar{f}(x)) = \text{Var}\left(\frac{1}{T} \sum_{t=1}^{T} f_t(x)\right) \] \[ \text{Var}(\bar{f}(x)) = \frac{1}{T^2} \left[ \sum_{t=1}^{T} \text{Var}(f_t(x)) + \sum_{t \neq s} \text{Cov}(f_t(x), f_s(x)) \right] \]

Using \(\text{Cov}(f_t, f_s) = \rho(x) \sigma^2(x)\):

\[ \begin{aligned} \text{Var}(\bar{f}(x)) &= \frac{1}{T^2} [T\sigma^2(x) + T(T-1)\rho(x)\sigma^2(x)] \\ &= \frac{\sigma^2(x)}{T^2}[T + T(T-1)\rho(x)] \\ &= \frac{\sigma^2(x)}{T} [1 + (T-1)\rho(x)] \\ &= \rho(x)\sigma^2(x) + \frac{1 - \rho(x)}{T} \sigma^2(x) \end{aligned} \]

Boosting

Assumption: Base models are weak classifiers (slightly better than random guessing).

In Boosting methods, each training sample is assigned a weight. T classifiers are learned iteratively.

After learning classifier \(h_t\), the weights are updated so that the subsequent classifier \(h_{t+1}\) focuses more on the training samples misclassified by \(h_t\).

The final classifier H integrates the predictions of each individual classifier. Traditionally requires setting an upper bound on the weak classifier error, though AdaBoost adapts.

AdaBoost — Adaptive Boosting

Uses different weighted versions of the same training set to train a set of weak classifiers, then integrates them in a weighted form to form a final strong classifier.
Input: Training set $ D = \{(x_1, y_1), \cdots, (x_n, y_n)\} $, where $ y_i \in \{-1, +1\} $, and a weak learning algorithm $ h $.

  • At each step of iteration, a weight \(w_1, w_2, \dots, w_n\) is assigned to the samples in the training set.

  • Initial sample weights are all equal, set to \(1/n\).

  • In each iteration, the weights of samples misclassified by the current weak classifier are increased, while the weights of correctly classified samples are decreased.

  • The weight of the weak classifier is determined by its weighted error rate. Error \(\epsilon_t\): "Sum of weights of misclassified samples" / "Total sum of weights of all samples in the training set" \[ \epsilon_t = \frac{\sum_{i=1}^n w_i^t I(y_i \neq h_t(x_i))}{\sum_{i=1}^n w_i^t} \] Weight of the weak classifier \(\alpha_t\) \[ \alpha_t = \frac{1}{2} \ln \left( \frac{1 - \epsilon_t}{\epsilon_t} \right) \]

    Relationship between error and weak classifier weight:

    • Smaller weak classifier error leads to larger weight.

    • Random guessing (error=0.5) yields weight 0.

    • Worse than random (error>0.5) yields negative weight.

    • In practice, we require weak classifiers with \(\alpha > 0\).

  • No need to pre-set an upper bound for the weak classifier error.

Algorithm Steps:

  1. Initialize sample weights \( w^1: w_i^1 = \frac{1}{n}, i = 1, 2, \cdots, n \); (all samples have the same initial weight)

  2. for \( t = 1 \) to \( T \) do

    1. Train a classifier \( h_t(x) \) using training dataset \( D \) with weights \( w^t \).

    2. Calculate the error of the current weak classifier: \[ \epsilon_t = \frac{\sum_{i=1}^n w_i^t I(y_i \neq h_t(x_i))}{\sum_{i=1}^n w_i^t} \]
    3. Based on the current weak classifier's error, calculate its weight: \[ \alpha_t = \frac{1}{2} \log \left( \frac{1 - \epsilon_t}{\epsilon_t} \right) \]
    4. Update sample weights to \( w^{t+1} \): (Increase the weight of samples misclassified by \( h_t(x) \)) \[ w_i^{t+1} \leftarrow \frac{w_i^t \cdot e^{-\alpha_t y_i h_t(x_i)}}{Z_t} \]

      Where $Z_t = \sum_{i=1}^n w_i^t \exp(-\alpha_t y_i h_t(x_i))$ is the normalization factor ensuring the new weights sum to 1.

  3. Output the final classifier: \( H(x) = \text{sign} \left( \sum_{t=1}^T \alpha_t h_t(x) \right) \).

Explanation of weight update:

  • When correctly classified, \( y_i h_t(x_i) = 1 \), the sample weight is multiplied by \(\exp(-\alpha_t) < 1\), decreasing the weight.

  • When misclassified, \( y_i h_t(x_i) = -1 \), the sample weight is multiplied by \(\exp(\alpha_t) > 1\), increasing the weight of misclassified samples.

Error Analysis

\[ \text{Error}_{\text{test}} = \mathbb{E}_{x,y} [L(y_i, H(x_i))] \]

(where L is a loss function) Theoretically, the upper bound of the generalization error may increase with T. However, empirical results show that even large T does not easily lead to overfitting.

Relationship: True generalization error $\le$ Training error + Complexity penalty term

Training Error:

  • The upper bound of the training error decreases as the number of iterations increases.

  • AdaBoost is often observed not to overfit even with many training iterations.

Ideal objective:

\[ \text{Error}_{\text{train}} = \frac{1}{N} \sum_{i=1}^N \mathbb{I}(H(x_i) \neq y_i) \]

Surrogate (exponential) objective minimized by AdaBoost:

\[ \text{Loss}_{\text{exp}} = \frac{1}{N} \sum_{i=1}^N \exp(-y_i f(x_i)) \]

Relationship (exponential loss upper bounds misclassification error):

\[ \mathbb{I}(H(x_i) \neq y_i) \leq \exp(-y_i f(x_i)) \]

where

\[ f(x) = \sum_{t=1}^T \alpha_t h_t(x) \] \[ H(x) = \text{sign}(f(x)) \]

Classification Margin

\[ \text{margin}(x, y) = y \cdot f(x) \]

  • Sign interpretation: \( yf(x) > 0 \) indicates correct classification; \( yf(x) < 0 \) indicates error.

  • Magnitude interpretation: \( |f(x)| \) represents classification confidence.

  • Physical interpretation: Minimizing exponential loss pursues not only correct classification but also a large margin.

Weak Learning Assumption

  • For binary classification, let the weighted error (error rate) of the weak classifier \( h_t \) at round t be \( \epsilon_t = \frac{1}{2} - \gamma_t \).

  • \( \gamma_t > 0 \) quantifies how much better \( h_t \) is than random guessing (1/2). It is called the edge.

  • To guarantee convergence, we require: \( \gamma_t > 0 \).

Significance: As long as the weak classifier is slightly better than random guessing, the algorithm works.

Theorem: Upper Bound on Training Error

Theorem: The training error of the final classifier \( H(x) \) produced by the AdaBoost algorithm satisfies:

\[ \frac{1}{N} \sum_{i=1}^{N} I(H(x_i) \neq y_i) \leq \frac{1}{N} \sum_{i=1}^{N} \exp(-y_i f(x_i)) = \prod_{t=1}^{T} Z_t \]

Derivation (unrolling the weight updates):

\[ w_i^{(T+1)} \prod_{t=1}^{T} Z_t = w_i^{(1)} \exp\left( -y_i \sum_{t=1}^{T} \alpha_t h_t(x_i) \right) = \frac{1}{N} \exp(-y_i f(x_i)) \]

Summing both sides over i (note \(\sum_i w_i^{(T+1)} = 1\)):

\[ \prod_{t=1}^{T} Z_t = \frac{1}{N} \sum_{i=1}^{N} \exp(-y_i f(x_i)) \]

Conclusion: The upper bound of the training error is the product of the normalization factors from each round.

Analyzing \(Z_t\):

\[ Z_t = \sum_{i=1}^{n} w_i^{(t)} e^{-\alpha_t y_i h_t(x_i)} \]

Partition samples into correctly classified set \( C_t = \{ i: y_i h_t(x_i) = +1 \} \) (weighted proportion \(1 - \epsilon_t\)) and misclassified set \( M_t = \{i: y_i h_t(x_i) = -1\} \) (weighted proportion \(\epsilon_t\)).

\[ Z_t = \sum_{i \in C_t} w_i^{(t)} e^{-\alpha_t} + \sum_{i \in M_t} w_i^{(t)} e^{\alpha_t} = (1 - \epsilon_t)e^{-\alpha_t} + \epsilon_t e^{\alpha_t} \]

Substitute the optimal \(\alpha_t = \frac{1}{2} \ln((1-\epsilon_t)/\epsilon_t)\), so \(e^{\alpha_t} = \sqrt{(1-\epsilon_t)/\epsilon_t}\) and \(e^{-\alpha_t} = \sqrt{\epsilon_t/(1-\epsilon_t)}\):

\[ Z_t = (1 - \epsilon_t)\sqrt{\frac{\epsilon_t}{1 - \epsilon_t}} + \epsilon_t\sqrt{\frac{1 - \epsilon_t}{\epsilon_t}} = \sqrt{(1 - \epsilon_t)\epsilon_t} + \sqrt{\epsilon_t(1 - \epsilon_t)} = 2\sqrt{\epsilon_t(1 - \epsilon_t)} \]

Let \(\epsilon_t = 1/2 - \gamma_t\):

\[ Z_t = 2\sqrt{\left(\frac{1}{2} - \gamma_t\right)\left(\frac{1}{2} + \gamma_t\right)} = 2\sqrt{\frac{1}{4} - \gamma_t^2} = \sqrt{1 - 4\gamma_t^2} \]

Using the inequality \( e^{-x} \geq 1 - x \) (or more precisely, \(\sqrt{1-4\gamma_t^2} \le e^{-2\gamma_t^2}\) for small \(\gamma_t\)):

\[ \sqrt{1 - 4\gamma_t^2} \leq \sqrt{e^{-4\gamma_t^2}} = e^{-2\gamma_t^2} \]

Thus, the product bound becomes:

\[ \prod_{t=1}^{T} Z_t = \prod_{t=1}^{T} 2\sqrt{\epsilon_t(1 - \epsilon_t)} = \prod_{t=1}^{T} \sqrt{1 - 4\gamma_t^2} \leq \prod_{t=1}^{T} e^{-2\gamma_t^2} = \exp\left(-2\sum_{t=1}^{T}\gamma_t^2\right) \]

Conclusion: As long as each weak classifier is better than random guessing (\(\gamma_t > 0\)), the training error of AdaBoost decreases exponentially fast towards zero as the number of rounds T increases. Advantages:

  • AdaBoost was originally intended to address underfitting (reduce bias), but in practice, due to the nature of ensemble learning, it also helps control overfitting (reduce variance).

  • Besides the number of weak classifiers (T), there are no additional hyperparameters to tune.

  • AdaBoost provides a framework; the weak classifier can be a decision tree or any other classification algorithm.

Disadvantages:

  • Due to integrating multiple weak classifiers, the interpretability of the model is reduced.

  • If the weak classifiers are too complex or perform too poorly, it can lead to overfitting.

  • Sensitive to outliers.

Unsupervised Learning

Types: Clustering, Dimensionality Reduction, Density Estimation, Anomaly Detection.

Clustering Models

Criteria (typically): Samples within the same cluster are close to each other; samples from different clusters are far apart.

K-means Clustering

  • Provide the desired number of clusters k.

  • Randomly select k instances as initial seeds, serving as the centroids for each cluster.

  • Iterate: Assign each instance to the cluster whose centroid is nearest (reassignment step); Recalculate the centroid of each cluster (update step).

  • Stop when convergence is reached (maximum iterations reached / centroids no longer change).

Objective: Minimize the sum of squared distances between each point and its assigned cluster centroid.

\[ \min_{\{C_k\}} \sum_{k=1}^{K} \sum_{x \in C_k} \| x - \mu^k \|^2 ,\quad \mu^k = \frac{1}{|C_k|} \sum_{x \in C_k} x \]

K-means guarantees convergence to a local optimum; finding the global optimum is NP-hard.

Time Complexity

  • Assume computing the distance between two instances is O(d), where d is the dimension of the vectors.

  • Reassignment: For each sample point, compute its distance to k centroids and select the nearest one. Complexity: O(knd).

  • Centroid calculation: Sum the vectors of each sample to its cluster's centroid and then average. Complexity: O(nd).

  • Assuming these two steps iterate l times: Total complexity O(lknd).

Choosing Seeds

  • Results vary depending on the random seed selection (sensitive to initialization).

  • Some seeds can lead to poor convergence speed or convergence to suboptimal clusters.

  • Use heuristics or other methods to select good seeds.

Choosing the value of k

Bayesian Information Criterion (BIC) can be used for model selection:

\[ \text{BIC}(M|X) = \text{LL}(X|M) - \frac{p}{2} \log(n) \]

Where p is the number of parameters in model M, and n is the number of samples. For example, with k=2 clusters in d dimensions: number of parameters = 2 centroids * d dimensions = 2d; BIC = log-likelihood - (d·log(n)). Advantages: Simple and intuitive to implement. Disadvantages: Requires specifying K beforehand; clustering results depend on the choice of initial K centroids; prone to getting stuck in local optima; does not handle non-globular cluster shapes well; sensitive to outliers.

Kernel-Kmeans

Extends K-means to non-linear cluster boundaries by mapping data to a high-dimensional feature space via a kernel function \(\phi\). The objective in feature space is:

\[ J = \sum_{k=1}^{K} \sum_{x_i \in C_k} \|\phi(x_i) - \mu_k^\phi\|^2 \]

Expanding the squared distance using the kernel trick:

\[ \|\phi(x_i) - \mu_k^\phi\|^2 = \langle \phi(x_i), \phi(x_i) \rangle - 2 \langle \phi(x_i), \mu_k^\phi \rangle + \langle \mu_k^\phi, \mu_k^\phi \rangle \] \[ \langle \phi(x_i), \phi(x_i) \rangle = K(x_i, x_i) \] \[ \langle \phi(x_i), \mu_k^\phi \rangle = \frac{1}{|C_k|} \sum_{x_j \in C_k} \langle \phi(x_i), \phi(x_j) \rangle = \frac{1}{|C_k|} \sum_{x_j \in C_k} K(x_i, x_j) \] \[ \langle \mu_k^\phi, \mu_k^\phi \rangle = \frac{1}{|C_k|^2} \sum_{x_j \in C_k} \sum_{x_l \in C_k} \langle \phi(x_j), \phi(x_l) \rangle = \frac{1}{|C_k|^2} \sum_{x_j \in C_k} \sum_{x_l \in C_k} K(x_j, x_l) \]

Thus, the distance can be computed solely using kernel evaluations:

\[ \|\phi(x_i) - \mu_k^\phi\|^2 = K(x_i, x_i) - \frac{2}{|C_k|} \sum_{x_j \in C_k} K(x_i, x_j) + \frac{1}{|C_k|^2} \sum_{x_a \in C_k} \sum_{x_b \in C_k} K(x_a, x_b) \]

EM Algorithm (Expectation-Maximization)

  • The EM algorithm is an iterative optimization strategy for parameter estimation in probabilistic models with latent variables

  • Widely used in: Gaussian Mixture Models (GMM), Hidden Markov Models (HMM), missing data imputation, etc.

Algorithm Steps

  1. E-step (Expectation step): Given the current parameter estimates, compute the expected value of the log-likelihood with respect to the posterior distribution of the latent variables \[ Q(\theta, \theta^{(t)}) = E_{Z|X,\theta^{(t)}}[\log P(X, Z | \theta)] \]
  2. M-step (Maximization step): Maximize the Q-function to update the parameters \[ \theta^{(t+1)} = \arg\max_{\theta} Q(\theta, \theta^{(t)}) \]

  3. Repeat E-step and M-step until convergence or reaching maximum iterations

EM in Gaussian Mixture Models (GMM)

  • GMM assumes data is generated by a mixture of K Gaussian distributions: \[ P(x) = \sum_{k=1}^{K} \pi_k \mathcal{N}(x | \mu_k, \Sigma_k), \quad \sum_{k=1}^{K} \pi_k = 1 \]
  • E-step: Compute the posterior probability (responsibility) that sample i belongs to component k \[ \gamma_{ik} = \frac{\pi_k \mathcal{N}(x_i | \mu_k, \Sigma_k)}{\sum_{j=1}^{K} \pi_j \mathcal{N}(x_i | \mu_j, \Sigma_j)} \]
  • M-step: Update the parameters using the responsibilities \[ \mu_k^{\text{new}} = \frac{\sum_{i=1}^{n} \gamma_{ik} x_i}{\sum_{i=1}^{n} \gamma_{ik}}, \quad \Sigma_k^{\text{new}} = \frac{\sum_{i=1}^{n} \gamma_{ik} (x_i - \mu_k^{\text{new}})(x_i - \mu_k^{\text{new}})^T}{\sum_{i=1}^{n} \gamma_{ik}} \] \[ \pi_k^{\text{new}} = \frac{1}{n} \sum_{i=1}^{n} \gamma_{ik} \]

Relationship between EM and K-means

  • K-means is a special case of the EM algorithm for GMM under the following conditions:

    • The covariance matrix of each Gaussian component is \(\Sigma_k = \epsilon I\) (isotropic)

    • \(\epsilon \to 0\) (approaches zero)

    • The responsibilities \(\gamma_{ik}\) degenerate to hard assignments (0 or 1)

  • K-means performs hard clustering, while EM performs soft clustering

Properties of the EM Algorithm

  • Advantages:

    • Can handle models with latent variables

    • Guarantees non-decreasing likelihood at each iteration

    • Relatively simple to implement and numerically stable

  • Disadvantages:

    • Sensitive to initialization and may converge to local optima

    • Slow convergence rate (linear convergence)

    • Requires knowledge of the latent variable distribution form

Hierarchical Clustering

Types (heuristic): Agglomerative (bottom-up, merging); Divisive (top-down, splitting).

Agglomerative Clustering

Start with each sample as its own cluster. At each iteration, merge the two most similar (closest) clusters. Repeat until all samples belong to a single cluster.

Time complexity: Finding the most similar clusters naively is O($n^2$) per step, and there are n steps, leading to O($n^3$). Improvement: Using a priority queue (min-heap) can reduce this to \(O(n^2 \log(n))\). For large n, one can first run K-means to get a moderate number of cluster centers, then perform hierarchical clustering on those centers.

Calculation of inter-cluster distance: Single-link (nearest neighbor), Complete-link (farthest neighbor), Average-link.

Divisive Clustering

Example: Bisecting K-means (similar to building a minimum spanning tree). Repeatedly select the cluster with the largest "radius" (e.g., highest SSE or diameter) and split it into two clusters using K-means (k=2).

Distance analysis method (a deterministic divisive approach):

  1. Find splitting point: Unlike K-means with random initialization, this method is deterministic. In the cluster \(C_{\text{old}}\) to be split, find the point \(u^*\) with the maximum average dissimilarity: \[ u^* = \arg \max_{i \in C_{\text{old}}} \left( \frac{1}{|C_{\text{old}}| - 1} \sum_{j \in C_{\text{old}}, j \neq i} d(i, j) \right) \]

    Initialize \(C_{\text{new}} = \{u^*\}\).

  2. Iterative splitting (e.g., using a "gravitational" or reassignment method): For each remaining point \(v\) in \(C_{\text{old}}\):

    • \( D_{\text{old}}(v) \): average distance from point \(v\) to points in \(C_{\text{old}}\).

    • \( D_{\text{new}}(v) \): average distance from point \(v\) to points in \(C_{\text{new}}\).

    Decision rule:

    • Compute the difference: \( \Delta(v) = D_{\text{old}}(v) - D_{\text{new}}(v) \).

    • If \( \Delta(v) > 0 \) (closer to \(C_{\text{new}}\) on average), transfer \(v\) from \(C_{\text{old}}\) to \(C_{\text{new}}\).

    • If \( \Delta(v) \leq 0 \) (closer to \(C_{\text{old}}\) on average), keep \(v\) in \(C_{\text{old}}\).

    Repeat the process (recalculating averages) until no points are transferred.

Characteristics of Hierarchical Clustering:

  • No explicit global objective function (unlike K-means).

  • Produces a full dendrogram (tree of cluster merges/splits), allowing any number of clusters to be obtained by cutting the tree at the desired level.

  • Cluster assignments for different numbers of clusters can be obtained without recomputation.

  • Single-link and complete-link represent extremes for cluster distance measures, both sensitive to outliers/noise.

  • Average-link offers a compromise, more robust to outliers.

  • Disadvantage: High computational cost. Additionally, partition decisions are irreversible (a sample assigned to a cluster at an early stage stays there).

Density-based Clustering: DBSCAN

Assumption: Clusters are high-density regions separated by low-density regions.

A dense cluster is a "density-connected" region where the point density is above a specified minimum.

\(\epsilon\)-neighborhood: The region centered at a sample with radius \(\epsilon\).

Core Object: A sample whose \(\epsilon\)-neighborhood contains at least \(N_{\min}\) samples (including itself).

Parameters: radius \(\epsilon\) and minimum number of samples \(N_{\min}\).

Definitions (assuming symmetric distance):

  • Directly density-reachable (asymmetric): \(x_j\) is within the \(\epsilon\)-neighborhood of core object \(x_i\).

  • Density-reachable (transitive): There exists a chain of samples \( p_1, p_2, \ldots, p_m \) with \( p_1 = x_i, p_m = x_j \), where each \( p_{k+1} \) is directly density-reachable from \( p_k \).

  • Density-connected (symmetric): Points \(x_i\) and \(x_j\) are density-connected if there exists a core point \(o\) such that both \(x_i\) and \(x_j\) are density-reachable from \(o\).

Advantages:

  • Does not require specifying the number of clusters.

  • Can discover clusters of arbitrary shape.

  • Effectively identifies noise points.

  • Insensitive to the order of samples (deterministic).

Disadvantages:

  • Parameters \(\epsilon\) and \(N_{\min}\) need tuning.

  • Performs poorly on datasets with varying densities.

Dimensionality Reduction Methods

Common linear dimensionality reduction methods: Principal Component Analysis (PCA); Linear Discriminant Analysis (LDA) Common non-linear dimensionality reduction methods: Multidimensional Scaling (MDS); Locally Linear Embedding (LLE)

Eigendecomposition

\[ A = U \Lambda U^{-1} \] \[ \Lambda = \text{diag}(\lambda_1, \lambda_2, \ldots, \lambda_n) = \begin{pmatrix} \lambda_1 & & \\ & \lambda_2 & \\ & & \ddots \\ & & & \lambda_n \end{pmatrix} \]

  • When \( A = A^T \) (symmetric, e.g., covariance matrix)

    • Eigenvalues are real.

    • Eigenvectors are mutually orthogonal.

    • U is orthogonal: \( U^{-1} = U^T \).

  • In this case, eigendecomposition is: \[ A = U \Lambda U^{-1} = U \Lambda U^T \]

Principal Component Analysis (Variance Maximization)

Properties:

  • Principal components are mutually orthogonal and uncorrelated, removing redundancy.

  • Sorted by descending variance; the first principal component contains the most information (variance), the second contains the next most, etc.

Algorithmic Principle:

Assume dataset \(D=[x_1, x_2, \ldots, x_n]\), each sample is a d-dimensional vector (continuous features).

Represent dataset as an \(n \times d\) matrix X.

Let \(W_{(d \times l)}\) be the transformation matrix that projects d-dimensional data to l dimensions (\(l < d\)). The projected data is \(Y = XW\).

The variance of the projected data Y (with columns centered) is proportional to:

\[ \text{var}(Y) = \frac{1}{n-1} \text{tr}(Y^T Y) = \frac{1}{n-1} \text{tr}(W^T X^T X W)= \text{tr}\left( W^T \frac{1}{n-1} X^T X W \right). \]

The covariance matrix of the original dataset is \(\Sigma = \frac{1}{n-1} X^T X\).

\[ \max_{W} \text{tr}(W^T \Sigma W) \]

s.t. \( w_i^T w_i = 1 \quad i \in \{1,2,\cdots,l\} \). (Orthonormality constraint)

\( w_i^T w_i = 1 \) (unit vector constraint: prevents variance from blowing up, standardizes projections).

\( w_i^T w_j = 0 \) for \(i \neq j\) (orthogonality constraint).

Role of orthogonality constraint: Prevents the optimization from selecting the same principal component repeatedly; forces each new PC to find maximum variance in a direction perpendicular to all previously found PCs, ensuring the resulting features are uncorrelated and have no information redundancy.

Note: Since \(\Sigma\) is symmetric, its eigenvectors are naturally orthogonal (\(w_i \perp w_j\)), so the orthogonality constraint is automatically satisfied if we use eigen-decomposition. However, it's listed for completeness in the Lagrangian formulation.

The maximum variance captured by PCA is the sum of the first l eigenvalues of the covariance matrix:

\[ \text{tr}(W^T \Sigma W) = \sum_{i=1}^{l} \lambda_i \]

where \(\lambda_i\) are the eigenvalues of \(\Sigma\) associated with the selected eigenvectors.

Lagrange multiplier method (for deriving PCA, considering orthonormality constraints explicitly):

\[ \mathcal{L}(W, \lambda) = \text{tr}(W^T \Sigma W) - \sum_{i=1}^{l} \lambda_i (w_i^T w_i - 1) \]

(Typically, symmetric \(\Sigma\) leads to eigenvectors satisfying orthogonality automatically via the symmetric eigenvalue problem. Including cross-orthogonality constraints would require more Lagrange multipliers.)

Derivative with respect to W (treating each \(w_i\) separately):

\[ \frac{\partial \mathcal{L}}{\partial w_i} = 2 \Sigma w_i - 2\lambda_i w_i = 0 \Rightarrow \Sigma w_i = \lambda_i w_i \]

PCA Algorithm: Input: Data matrix X, target reduced dimension l. Output: Transformation matrix \(W = (w_1, w_2, \ldots, w_l)\).

  1. Center each sample \(x_i\) in X: \[ x_i \leftarrow x_i - m, \]

    where \(m = \frac{1}{n} \sum_{j=1}^n x_j\) is the sample mean vector.

  2. Compute the covariance matrix \(\Sigma = \frac{1}{n-1} X^T X\).

  3. Perform eigendecomposition on \(\Sigma\) and sort eigenvalues in descending order: \(\lambda_1 \geq \lambda_2 \geq \ldots \geq \lambda_d\).

  4. Select the eigenvectors corresponding to the largest l eigenvalues to form the transformation matrix W.

Relationship between covariance matrix \(X^T X\) and Gram matrix \(X X^T\): If \(u_k\) is an eigenvector of \(X X^T\) with eigenvalue \(\lambda_k\), then the corresponding eigenvector for \(X^T X\) is \(w_k = \frac{X^T u_k}{\sqrt{\lambda_k}}\). Conversely, \(u_k = \frac{X w_k}{\sqrt{\lambda_k}}\).

Variance retained ratio (for choosing l):

\[ \text{varratio}(l) = \frac{\sum_{i=1}^l \lambda_i}{\sum_{j=1}^d \lambda_j} \]

Essence: Features in the low-dimensional space are linear combinations of the original high-dimensional features.

PCA has no hyperparameters to tune (except l, which can be chosen via variance ratio) and finds the global optimum (since it's based on eigenvalue decomposition).

Time complexity is O(\(d^3\)), space complexity is O(\(d^2\)) for the covariance matrix.

Linear Discriminant Analysis LDA / Fisher Discriminant Analysis (Supervised)

Dataset is \(D=\{(x_1,y_1),(x_2,y_2),...,(x_n,y_n)\}\), sample labels \(y_i\) take values in \(\{1,2,...,C\}\). Number of samples in class c is \(n_c\) (\(n = \sum_{c=1}^C n_c\)). Mean of samples in class c: \(m_c = \frac{1}{n_c} \sum_{i=1}^{n_c} x_i\). Overall mean: \(m = \frac{1}{n} \sum_{i=1}^n x_i\). Projected data: \(Z = X W\) (where \(z_i\) is the projection of \(x_i\)).

Within-class scatter (measuring dispersion inside classes): Before projection: Scatter matrix for class c: \(S_c = \sum_{i=1}^{n_c}(x_i - m_c)(x_i - m_c)^T\). Total within-class scatter matrix (weighted average): \(S_w = \sum_{c=1}^C \frac{n_c}{n} S_c\). After projection onto direction w: Within-class scatter (scalar): \(w^T S_w w\).

Between-class scatter (measuring separation between classes): Before projection: Between-class scatter matrix: \(S_b = \sum_{c=1}^C \frac{n_c}{n}(m_c - m)(m_c - m)^T\). After projection onto direction w: Between-class scatter (scalar): \(w^T S_b w\).

Optimization objective for Fisher's LDA (maximize ratio of between-class to within-class scatter):

$$\max_w \frac{w^T S_b w}{w^T S_w w}$$

Equivalent constrained formulation: \(\max_w w^T S_b w\) s.t. \(w^T S_w w = 1\).

Lagrange multiplier method: \(\mathcal{L}(w, \lambda) = w^T S_b w - \lambda (w^T S_w w - 1)\).

Take derivative w.r.t w and set to zero:

\[ \frac{\partial \mathcal{L}(w, \lambda)}{\partial w} = 2S_b w - 2\lambda S_w w = 0, \] \[ S_b w = \lambda S_w w \quad \Rightarrow \quad S_w^{-1} S_b w = \lambda w \]

Thus, the optimal projection directions w are the eigenvectors of \(S_w^{-1} S_b\). The corresponding eigenvalue \(\lambda\) equals the objective ratio \(w^T S_b w / (w^T S_w w)\). The eigenvector with the largest eigenvalue gives the best single direction.

Differences between LDA and PCA:

  • Basic idea:

    • PCA selects directions that maximize the variance of the projected data, preserving the most information (in terms of variance) from the original data.

    • LDA uses label information, maximizing the distance between different classes' means while minimizing the variance within each class after projection.

  • Learning paradigm:

    • PCA is unsupervised, applicable more broadly, but the resulting low-dimensional representation may not be optimal for classification.

    • LDA is supervised, capable of both dimensionality reduction and classification (though it is not a classifier by itself; it finds discriminative directions).

Classical Multidimensional Scaling (MDS)

Scenario: Cannot directly observe sample features, only the distances or similarities between samples.

Goal: Find low-dimensional coordinates \(z_i\) that preserve the given distances \(d_{ij}\) as well as possible.

\[ \min_{z_1, z_2, \cdots, z_n} \sum_{i \neq j} (\|z_i - z_j\|_2 - d_{ij})^2 \]

Assume the original (unknown) high-dimensional data points \(x_i\) are centered at the origin: \(\sum_{i=1}^n x_i = 0\). Let the inner product matrix be \(B = X X^T\), where \(b_{ij} = x_i^T x_j\).

Relating squared Euclidean distances to inner products:

\[ d_{ij}^2 = \|x_i - x_j\|^2 = (x_i - x_j)^T (x_i - x_j) = x_i^T x_i + x_j^T x_j - 2x_i^T x_j = b_{ii} + b_{jj} - 2b_{ij}. \]

Summing over j:

\[ \sum_{j=1}^n d_{ij}^2 = \sum_{j=1}^n b_{ii} + \sum_{j=1}^n b_{jj} - 2\sum_{j=1}^n b_{ij} \]

First term: \(\sum_{j=1}^n b_{ii} = n b_{ii}\). Second term: \(\sum_{j=1}^n b_{jj} = T\) (define \(T = \sum_{j=1}^n b_{jj}\)). Third term: \(\sum_{j=1}^n b_{ij} = x_i^T (\sum_{j=1}^n x_j) = x_i^T \cdot 0 = 0\) (due to centering). Thus:

\[ \sum_{j=1}^n d_{ij}^2 = n b_{ii} + T. \]

Similarly, summing over i:

\[ \sum_{i=1}^{n} d_{ij}^2 = n b_{jj} + T. \]

Summing over both i and j:

\[ \sum_{i=1}^{n} \sum_{j=1}^{n} d_{ij}^2 = \sum_{i=1}^{n} (n b_{ii} + T) = n \sum_{i=1}^{n} b_{ii} + n T = n T + n T = 2n T. \]

Hence:

\[ T = \frac{1}{2n} \sum_{i=1}^{n} \sum_{j=1}^{n} d_{ij}^2. \]

Now solve for \(b_{ij}\) in terms of the squared distances. From \(d_{ij}^2 = b_{ii} + b_{jj} - 2b_{ij}\), we have:

\[ b_{ij} = -\frac{1}{2}(d_{ij}^2 - b_{ii} - b_{jj}). \]

Substitute the expressions for \(b_{ii}\) and \(b_{jj}\) derived from \(\sum_j d_{ij}^2 = n b_{ii} + T\) and \(\sum_i d_{ij}^2 = n b_{jj} + T\):

\[ b_{ii} = \frac{1}{n} \sum_{j=1}^n d_{ij}^2 - \frac{T}{n}, \quad b_{jj} = \frac{1}{n} \sum_{i=1}^n d_{ij}^2 - \frac{T}{n}. \]

Plugging back and simplifying yields the standard formula for \(b_{ij}\):

\[ b_{ij} = -\frac{1}{2} \left( d_{ij}^2 - \frac{1}{n} \sum_{j=1}^n d_{ij}^2 - \frac{1}{n} \sum_{i=1}^n d_{ij}^2 + \frac{1}{n^2} \sum_{i=1}^n \sum_{j=1}^n d_{ij}^2 \right) \]

Or equivalently:

\[ b_{ij} = \frac{1}{2} \left( \frac{1}{n} \sum_{i=1}^{n} d_{ij}^2 + \frac{1}{n} \sum_{j=1}^{n} d_{ij}^2 - \frac{1}{n^2} \sum_{i=1}^{n} \sum_{j=1}^{n} d_{ij}^2 - d_{ij}^2 \right) \]

MDS Algorithm: Input: Distance matrix \(D\) (size n x n), target reduced dimension l. Output: Reduced \(n \times l\) data matrix \(Z\).

  1. Compute matrix \(B\) from the distance matrix \(D\) using the formula above for \(b_{ij}\).

  2. Perform eigendecomposition of \(B\): \(B = U \Lambda U^T\).

  3. Select the l largest eigenvalues to form diagonal matrix \(\Lambda_l\), and the corresponding eigenvectors to form \(U_l\) (columns).

  4. The reduced coordinates are given by \(Z = U_l \Lambda_l^{1/2}\). (Since \(B = X X^T = U \Lambda U^T\), setting \(X = U \Lambda^{1/2}\) works; retaining top l components gives the best l-dimensional Euclidean representation.)

Locally Linear Embedding (LLE)

Manifold: A space that is locally Euclidean, a generalization of curves and surfaces in Euclidean space.

Assumption: High-dimensional data lies on or near a low-dimensional manifold. Euclidean distances can be misleading globally.

Manifold dimension: The minimum number of coordinates needed to uniquely identify points on the manifold, often much lower than the embedding dimension.

LLE: Reduces data to a lower dimension while preserving local linear relationships (reconstruction weights).

Steps:

  • Select neighbors. For each data point \(x_i\), find its k nearest neighbors (typically \(k \leq d\), the original dimension).

  • Local linear reconstruction. Find weights \(W \in \mathbb{R}^{n \times n}\) (where \(w_{ij}\) is the contribution of point j to reconstructing point i) that minimize the reconstruction error, subject to the sum of weights for each i being 1: \[ \min_{W} \sum_{i=1}^{n} \| x_i - \sum_j w_{ij} x_j \|_2^2, \quad \text{s.t. } \sum_{j} w_{ij} = 1 \text{ for each } i. \]

    (Only neighbors j have non-zero \(w_{ij}\)).

  • Find low-dimensional embedding. Using the same weights \(W\), find low-dimensional coordinates \(Y\) (points \(y_i\)) that minimize a similar reconstruction error in the low-dimensional space: \[ \min_{Y} \sum_{i=1}^{n} \| y_i - \sum_{j} w_{ij} y_j \|_2^2 \]

    (This leads to an eigenvalue problem.)

Advantages:

  • LLE has only two main hyperparameters to tune: the number of neighbors \(k\) and (implicitly) a regularization parameter \(\lambda\) (often used when solving the linear systems).

  • LLE finds a global optimum solution (via solving a sparse eigenvalue problem), not prone to local minima issues (unlike some iterative methods).

Disadvantages:

  • Since it relies on KNN to find neighbors, if the sample size is too small, neighbors may be far away, degrading the embedding quality.

  • LLE is sensitive to noise and outliers, which can disrupt the local linearity assumption.

  • No explicit mapping function (unlike PCA or LDA which provide a projection matrix \(W\) for new samples; for LLE, new samples would require solving an optimization or using out-of-sample extensions).

Other Dimensionality Reduction Methods

Kernel PCA (KPCA)

KPCA uses the kernel trick to perform PCA in a high-dimensional (possibly infinite) feature space implicitly. The covariance matrix in feature space cannot be explicitly computed, so KPCA works with the Gram matrix (kernel matrix).

Centered feature map values:

\[ \tilde{\Phi}(x_i) = \Phi(x_i) - \frac{1}{n} \sum_{j=1}^{n} \Phi(x_j) \]

Corresponding centered kernel matrix \(\tilde{K}\):

\[ \tilde{K}_{ij} = \langle \tilde{\Phi}(x_i), \tilde{\Phi}(x_j) \rangle = \tilde{\Phi}(x_i)^T \tilde{\Phi}(x_j) \] \[ \tilde{K}_{ij} = \langle \Phi(x_i), \Phi(x_j) \rangle - \langle \Phi(x_i), \mu_{\Phi} \rangle - \langle \mu_{\Phi}, \Phi(x_j) \rangle + \langle \mu_{\Phi}, \mu_{\Phi} \rangle \]

where \(\mu_{\Phi} = \frac{1}{n} \sum_{k=1}^{n} \Phi(x_k)\).

The terms can be expressed using the original kernel matrix K:

\[ \langle \Phi(x_i), \mu_{\Phi} \rangle = \frac{1}{n} \sum_{k=1}^{n} \langle \Phi(x_i), \Phi(x_k) \rangle = \frac{1}{n} \sum_{k=1}^{n} K_{ik} = \frac{1}{n} (K \mathbf{1}_n)_i \] \[ \langle \mu_{\Phi}, \mu_{\Phi} \rangle = \frac{1}{n^2} \sum_{k=1}^{n} \sum_{l=1}^{n} \langle \Phi(x_k), \Phi(x_l) \rangle = \frac{1}{n^2} \sum_{k=1}^{n} \sum_{l=1}^{n} K_{kl} = \frac{1}{n^2} \mathbf{1}_n^T K \mathbf{1}_n \]

Thus, the centered kernel matrix can be computed directly from K. KPCA then performs standard PCA (eigendecomposition) on \(\tilde{K}\) to find principal components in the feature space, which correspond to non-linear components in the input space.

Isometric Mapping (Isomap)

Isomap is an extension of MDS that attempts to preserve geodesic (manifold) distances instead of straight-line Euclidean distances. It approximates geodesic distances by computing shortest paths on a graph built from KNN connections in the input space.

Advantage: Isomap can better preserve the global structure of the manifold (e.g., unrolling a Swiss roll).

Disadvantage: Sensitive to noise (which can create "shortcuts" in the graph) and has relatively high computational complexity (shortest path computation is O($n^2$ log n + $n^2$ k) or similar).
